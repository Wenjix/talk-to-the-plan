## Design Context

### Users
Solo knowledge workers and team facilitators who brainstorm, research, and build plans through AI-powered exploration. They come with a topic or question and want to explore it from multiple angles using distinct AI personas. Context ranges from individual deep thinking to collaborative planning sessions.

### Brand Personality
**Thoughtful, bold, playful.**
Serious about ideas but not stuffy. The tool has energy and personality — the four AI personas (Expansive, Analytical, Pragmatic, Socratic) are characters, not just labels. Exploration should feel dynamic, not clinical.

### Aesthetic Direction
**Expressive creative tool** — inspired by Figma's creative energy and Linear's clean execution.

- Personas have visual presence through their signature colors (purple, blue, teal, red)
- The canvas is the star — UI chrome should support, not compete
- Bold color use is encouraged where it carries meaning (persona identity, status, promotion)
- Dark mode is a first-class experience, not an afterthought

**References:** Linear (clean, fast, keyboard-driven, restrained accents), Figma (bold identity, creative energy, professional but playful)

**Anti-references:** Generic ChatGPT chat-thread UIs, enterprise dashboards (Jira-gray), skeuomorphic/retro design

### Design System

**CSS Architecture:** CSS Modules + CSS custom properties. Theme file at `src/components/Settings/theme.css`. Dark mode via `[data-theme="dark"]` attribute.

**Color Palette:**
- Brand accent: `#6366f1` (indigo), hover `#4f46e5`
- Dark mode accent: `#818cf8` (lighter indigo)
- Persona colors: Expansive `#7B4FBF`, Analytical `#4A90D9`, Pragmatic `#3DAA6D`, Socratic `#D94F4F`
- Status: success `#22c55e`, error `#ef4444`, warning `#f59e0b`, promotion `#d4a017`

**Typography:** System font stack (`system-ui, Avenir, Helvetica, Arial, sans-serif`). Weights 400/500/600/800. Sizes range 0.7rem (badges) to 1.1rem (headings). No custom web fonts.

**Spacing:** Empirical scale — 4/6/8/12/16/20px. Standard gap: 8px. Standard padding: 16px.

**Border Radius:** 4px (badges), 6px (buttons/inputs), 8px (dropdowns), 12px (cards/modals).

**Transitions:** 0.15s standard, 1.5-2s pulse animations for status indicators.

**Layout:** 48px toolbar, 280-360px card width, 380px sidebar panels, flex-based responsive layout.

### Design Principles

1. **Personas are characters** — Each persona's color is its identity. Use it consistently and boldly in dots, borders, backgrounds, and accents. Users should recognize a persona by color alone.

2. **Canvas first** — The exploration graph is the primary interface. All chrome (toolbar, panels, settings) should feel lightweight and recessive until needed. Never let UI compete with content.

3. **Motion with purpose** — Animations signal state (generating, ready, recording) not decoration. Pulse means "working," slide means "arriving," scale means "interactive." No animation without a reason.

4. **Density when exploring, clarity when reading** — The canvas can be information-dense during exploration. But individual cards, plans, and dialogue panels should have generous breathing room.

5. **Light mode is default, dark mode is polished** — Light mode is the primary experience and where most users start. Dark mode should be equally well-maintained and feel intentional, not an afterthought.

### Accessibility
- Target WCAG AA compliance
- Maintain 4.5:1 contrast ratio for text, 3:1 for large text and UI components
- Keyboard navigation for all interactive elements
- Respect `prefers-reduced-motion` for pulse/slide animations
